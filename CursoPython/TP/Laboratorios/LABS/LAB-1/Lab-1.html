Qu es una API?

Las APIs (Application Programming Interfaces, o Interfaces de Programacin de Aplicaciones) son un conjunto de definiciones y protocolos que permiten que las diferentes aplicaciones de software se comuniquen entre s.

API (Interface de Programacin de Aplicaciones): Es una interfaz que define las interacciones entre diferentes piezas de software. Una API especifica cmo deben interactuar los componentes del software entre s. Esto incluye qu funciones estn disponibles, cmo se llaman esas funciones, qu parmetros se deben proporcionar y qu resultados se pueden esperar.

Componentes de una API:

No se preocupen, las veremos ms a fondo en proximos modulos!

  • Endpoints
  • Mtodos HTTP
  • Headers
  • Body

Ejemplo de uso de una API:

Imagina que tienes una aplicacin de clima que necesita obtener los datos meteorolgicos de diferentes ciudades. En lugar de tener una base de datos propia con esta informacin, tu aplicacin puede usar una API pblica de un servicio meteorolgico. El proceso sera algo as:

  • Tu aplicacin enva una solicitud a la API del servicio meteorolgico en un endpoint como https://api.weather.com/v3/weather/forecast?city=Barcelona.
  • La API del servicio meteorolgico procesa la solicitud, recupera los datos del clima actual de su base de datos y enva una respuesta con esta informacin.
  • Tu aplicacin recibe la respuesta y presenta los datos al usuario, como la temperatura actual, la previsin del tiempo, etc.

Beneficios de las APIs

  • Modularidad: Las APIs permiten dividir aplicaciones grandes en mdulos ms pequeos y manejables, que pueden ser desarrollados y mantenidos de forma independiente.

  • Reusabilidad: El cdigo que expone una API puede ser reutilizado por mltiples aplicaciones, reduciendo la duplicacin de esfuerzos.

  • Interoperabilidad: Las APIs permiten que diferentes sistemas y tecnologas se comuniquen entre s, independientemente del lenguaje de programacin en el que estn escritos.

  • Escalabilidad: Facilitan el escalado de aplicaciones, ya que los servicios pueden ser distribuidos en diferentes servidores.

2- Realizar un programa que escuche nuestra voz y la transcriba a texto utilizando la libreria SpeechRecognition

#Como se usa SpeechRecognition?
import speech_recognition as sr
r =sr.Recognizer()

with sr.Microphone() as source:
    print("Hable: ")
    audio = r.listen(source)
    try:
        text = r.recognize_google(audio, language='es-ES')
        print(f"Dijiste: {text}")
    except sr.UnknownValueError:
        print("Lo siento, no he podido entender lo que has dicho.")
    except sr.RequestError:
        print("Lo siento, ha habido un problema al intentar comunicarme con el servicio de Google.")

Repasando algunas lineas de codigo:

La palabra reservada as sirve para asignarle el nombre a una biblioteca importada, en este caso le asignamos sr a speech_recognition para que nos sea mas comodo trabajar.

import speech_recognition as sr

La palabra reservada whit en Python se usa para simplificar el manejo de recursos que necesitan ser configurados y limpiados adecuadamente, como archivos, conexiones de red, o en este caso, el micrfono. La principal ventaja de usar with es que asegura que los recursos sean liberados correctamente despus de su uso, incluso si ocurre un error durante su uso.

with sr.Microphone() as source:

El bloque try except se usa para manejar errores que pueden ocurrir durante la ejecucin del cdigo.

  • try: Intenta ejecutar el cdigo dentro de este bloque. Si todo funciona bien, se ejecuta sin problemas.

  • except sr.UnknownValueError: Si ocurre un error del tipo UnknownValueError, significa que el reconocedor de habla no pudo entender el audio. En este caso, se ejecuta el siguiente bloque de cdigo.

  • except sr.RequestError: Si ocurre un error del tipo RequestError, significa que hubo un problema al intentar comunicarse con el servicio de Google. En este caso, se ejecuta el siguiente bloque de cdigo.

    try:
        text = r.recognize_google(audio, language='es-ES')
        print(f"Dijiste: {text}")
    except sr.UnknownValueError:
        print("Lo siento, no he podido entender lo que has dicho.")
    except sr.RequestError:
        print("Lo siento, ha habido un problema al intentar comunicarme con el servicio de Google.")
